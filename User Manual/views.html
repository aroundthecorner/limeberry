Views

Views are html pages that can be loaded by a controller, you can not directly redirect your application to a view file.
limeberry stores view files in a folder called 'view'. You can create sub folders in view folder if you want to seperate parts of your application.


Render()

Return a view to user. 

Render(string  $prm_file,boolean  $isusual  = true);

$prm_file File name of your view file
$isusual *optional. If your view is not located in 'view' directory.


isAvailable()

Return true if view file available 

isAvailable(string  $prm_file,boolean  $isusual  = true); : boolean

$prm_file File name of your view file
$isusual *optional. If your view is not located in 'view' directory.

Example Use 

$this->View->isAvailable("home\index.phtml");
                    

$this->View->isAvailable("test_folder\index.html", false);
                    

if($this->View->isAvailable("home/home.phtml"))
{
    $this->View->Render("home/home.phtml");
}else{
    echo "Page not found";
}



Transfer Variables Between Controllers & Views

Let's say you have a indexController.php controller in your controller directory, and you want to set a variable in your view file. There is lots of methods to do that like ObjectMapper of limeberry Franework, but in classic way you need to define a variable in your view file; 


...HTML CODES
         < ?php echo $this->sayhello; ?> 
...HTML CODES


And in your controller 
use limeberry\Controller As plController;

class indexController extends plController{

    
    public function IndexAction($param=0)
    {
            $this->View->sayhello = "Hello World :)"; 
            $this->View->Render("test.phtml");
    }
}
