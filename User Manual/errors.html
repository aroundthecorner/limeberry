Manage Errors



Error Handling

By default limeberry framework has Error Handling controller located in 'controller' folder. This class contains a NotFound action which returns simply a string message to screen.

If you desire to use a view file to show a 404 error you can design a view file and simply change the command on NotFount to render this view.

Expand Error Handlings

If you want to expand ErrorHandling class you can add your own actions redirect from other controllers when you need.
Assume that we have a calculator application and a calc.php controller. If user input invalid data to calculator you can redirect to your own error action.
For example;
We post data to controller from view to calc.php controller. User input is not integer and we want to warn user about that. in controller you can type: 

use limeberry\url;
header('location: '.url::redirectToAction("ErrorHandling", "InvalidInput"));

in ErrorHandling.php 
   public function InvalidInputAction()
   {
        echo "This input type is not supported";
   }
