    3. Directory Structure


    configThis folder is used to store other configuration files of your web application.

    controllerAll your controllers should be in this folder. When you make a request, according to your url the framework will find a controller to respond.

    librarySometimes you will need to use third party codes or your own classes. In this case you must put all third party libraries here to include them in your code. limeberry has a function to load libraries when you need.

    templateTlimeberry framework has a templating engine. if you want to avoid code repetition you can create a template and use them with your views.All your html templates must be located in this folder. The 'master' folder contains two other sub folders css(for css files) and script(for javascript). Please read topic related to template engine for getting more information.

    modelThis folder is used to store model classes.

    limeberryAll files and classes of limeberry framework can be found here.

    viewview files of your application must be in this folder. You can use sub folders to separate the parts of your website.

    outputThis folder contains 3 text files. This files are automatically generated by your code and holds the Errors, Warnings and output messages.

    index.phpThis file is the most important file of your application, we can call it entry point for your application. All configurations and url analyzes are made here for first time.

    app_config.phpThis files contains some configurations of your project

    NOTE 
    All folders contains (and should contain) a index.html file to prevent folder listing.

    URL Structure

    In limeberry framework urls are generated automatically. If you want to use routes the only thing you need is call a redirection function.

    http://www.examplesite.com/[controller]/[function]/[parameters]

    Example Controller 

    use limeberry\controller As p_ctrl;
    
class indexController extends p_ctrl
{
    public function indexAction($parameter=0)
    {       
        echo "Hello ".$parameter." Times from Index Action";
    }
}
    For example if we navigate to http://examplesite.com/Index/Index the IndexAction will be return Hello 0 Times from Index Action. For http://examplesite.com/Index/Index/seven the IndexAction will be return Hello seven Times from Index Action.
